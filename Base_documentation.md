## 4) Text Format

### Headings
- Use `#` for main heading  
- Use `##` for subheading  
- Use `###` for smaller heading  

### Bold Text
- Use `**text**`  
Example: **sherab**

### Italic Text
- Use `*text*`  
Example: *sherab*

### Bold + Italic Text
- Use `***text***`  
Example: ***sherab***

---

## 5) Version Control
Version control is a system that tracks changes made to files over time. It allows you to:
- Revisit previous versions  
- Collaborate without overwriting others’ work  
- Maintain a clear history of your project  

---

## 6) Committing and Saving Changes
After editing the documentation, the next step is to save your work using version control.

### Steps:
- Click **Commit** in the Web IDE after making changes  
- A message box will appear  
- Write a clear commit message (e.g., "Added documentation images")  
- Click **Commit and Push to 'main'**  

<img src="https://drive.google.com/thumbnail?id=1V3u8VBQ7wIZbsj7x9z1wRM252usnO5zY&sz=w800" alt="Commit Step 1">

After clicking, a text box will appear. Write what you have done (for example: added photos in base documentation).
